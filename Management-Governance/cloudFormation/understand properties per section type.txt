-AWS::Lambda::Function
  -Code
    -code for the function
  -Environment/Variables
    -Environment variables that are accessible from function code during execution
    -Custom and pre-defined environment variables can be passed in
    -Custom environment variables can be set in the YAML file 
  -Handler
    -The name of the method within your code that Lambda calls to run your function
    -Handler is required if the deployment package is a .zip file archive
  -Properties
    -CodeUri
    -FunctionName
    -Handler
    -Runtime
      -identifier of the function's runtime
      -Runtime is required if the deployment package is a .zip file archive
  -Role
    -ARN of the function's execution role
  -Timeout
    -amount of time (in seconds) that Lambda allows a function to run before stopping it
    -default is 3 seconds
    -max allowed value is 900 seconds
  -TracingConfig
  -VpcConfig
    -SecurityGroupIds
      -Needed when Lambda function needs to be in same security group as a service that will interact with it
    -SubnetIds
      -Needed when Lambda function needs to be in same subnet as a service that will interact with it
    -ref @ https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-properties-lambda-function-vpcconfig.html
  -ref @ 
    -https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-resource-lambda-function.html#aws-resource-lambda-function-properties
