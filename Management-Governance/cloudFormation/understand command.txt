command
-!FindInMap
  -returns the value corresponding to keys in a two-level map that's declared in the Mappings section
  -ref @ https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-findinmap.html
-!GetAtt
  -returns the value of an attribute from a resource in the template
  -Even if the attribute is auto created when the object is created in aws (like an ARN for a SNS Topic), the attribute needs to have been specified by you in the cloudformation template
  -Use !Ref command instead when you need to reference an attribute that is not specified by you in the cloudformation template and the attribute is the default attribute pulled by !Ref
  -ref @ 
    -https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-getatt.html
    -https://theburningmonk.com/cloudformation-ref-and-getatt-cheatsheet/
-!ImportValue
  -returns the value of an output exported by another stack. You typically use this function to create cross-stack references
  -ref @ 
    -https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-importvalue.html
    -https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/outputs-section-structure.html
-!Ref
  -Reference other blocks already in the .YAML file without having to write it again
  -returns the value of the specified parameter or resource
  -!Ref returns only the default attribute for another resources, while !GetAtt can return any attribute for the other resource 
  -ref @ 
    -https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-ref.html
    -https://theburningmonk.com/cloudformation-ref-and-getatt-cheatsheet/
-!Select
  -returns a single object from a list of objects by index
  -ref @ https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-select.html
-!Sub
  -Can be used for resource names that have variables
  -ref @ https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/intrinsic-function-reference-sub.html
